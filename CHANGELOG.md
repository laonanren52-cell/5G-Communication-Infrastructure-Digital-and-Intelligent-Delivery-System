# 版本记录

本文记录 `5G通信基建数智化交付系统 (Demo 版)` 到目前为止的主要改版过程。版本号按本项目在当前工作流中的功能演进顺序整理。

## 第 1 版：基础 Streamlit Demo

- 建立单页面 Streamlit 应用骨架。
- 支持内置基站设计元数据样例。
- 支持 Excel/CSV 上传和 Pandas 数据预览。
- 建立必填字段校验逻辑。
- 建立基础 BOM 规则估算函数。
- 保留本地 Mock 报告生成，保证无 API Key 时可演示。

## 第 2 版：工程报告与真实下载

- 扩展本地 Mock 报告内容，使其更接近通信施工交付文档。
- 报告结构覆盖工程概况、BOM、工序指导、安全注意事项、质量验收、风险闭环建议。
- 增加 Word 可打开的 `.doc` 报告下载。
- 保留 Demo 版说明：Word 文件为 HTML 格式 `.doc`，不是正式 `.docx`。

## 第 3 版：外接大模型 API 与大文件处理

- 增加 OpenAI 兼容 Chat Completions API 调用。
- 增加 API 地址、模型名称、API Key、超时时间和样本行数配置。
- 大型工程表先压缩为工程摘要，再附带少量样本行给模型。
- API 失败或超时时可自动回退本地规则报告。
- 增加生成过程进度反馈，避免长时间调用时页面无状态。

## 第 4 版：SaaS 控制台视觉改版

- 将页面整体调整为浅色 B2B SaaS 控制台风格。
- 左侧侧边栏改为浅色控制台。
- 顶部增加产品工作台、状态条和流程说明。
- 数据预览与数据质量校验改为双栏面板。
- 增加规则指标总览：记录数、线缆总长、字段数、生成模式。

## 第 5 版：Taste Skill Dashboard 报告预览

- 根据 Taste Skill 的 dashboard 方向拆分报告预览。
- 新增工程概况、BOM、质量验收、完整报告页签。
- 工程概况使用指标和重点信息模块呈现。
- BOM 使用结构化表格呈现物料名称、规格、估算数量、单位和施工用途。
- 质量验收使用检查清单呈现设备安装、线缆工艺、电源接入、接地、光链路和资料闭环。

## 第 6 版：修复 HTML 露出与默认数据预览

- 修复自定义 HTML 拼接导致 `<div class=...>` 代码显示在页面上的问题。
- 报告预览改用 Streamlit 原生组件，避免 HTML 解析失败。
- 未点击生成报告时，也展示基于当前数据的工程概况、BOM 和质量验收预览。
- 点击生成后再替换为正式报告并开放下载。

## 第 7 版：中文化高端 UI 与双下载

- 根据新的前端审美规范，将页面进一步中文化。
- 品牌名重设计为“云筑交付中枢”。
- 顶部导航、状态条、bento 工作流模块全部改为中文。
- “查看工程工作台”和“生成交付报告”改为真实可交互按钮。
- 左侧控制台改为可收纳分组：数据接入、生成设置、真实 API 配置、执行。
- Word 导出改为真实 HTML 表格结构，避免在 Word 中显示 Markdown 表格源码。
- 新增 Excel `.xlsx` 交付清单下载，包含工程概况、BOM物料清单、质量验收、原始设计数据、完整报告多个 Sheet。

## 第 8 版：视觉细节修正

- 移除标题中的内嵌小图片，避免生成横向白条。
- 移除外部随机占位图背景，改为稳定 CSS 渐变质感背景。
- 修复顶部按钮玻璃外壳造成的横向白条。
- 调整 Hero 说明文字居中显示。
- 增加顶部导航高度和页面上边距，避免导航文字裁切。
- 保持页面可访问性和 Streamlit 单页 Demo 架构不变。

## 第 9 版：融合赛道 3 的轻量智能审查

- 在赛道 4 主流程前增加“轻量智能审查结果”模块。
- 新增 `run_design_review(df)`、`build_review_summary(review_df, total_rows)`、`render_review_result(review_df, summary)`。
- 审查内容覆盖字段完整性、线缆距离异常、站点编号重复、纤芯占用异常、取电方式施工风险和资源配置冲突。
- 审查结果统一输出 `check_item`、`level`、`message`、`field`、`row_index`、`site_id`、`suggestion` 等字段。
- 页面展示总体审查结论、总记录数、错误数量、高风险数量、提醒数量和正常数量。
- 当存在错误级问题时，系统仍允许查看 BOM 和报告预览，但阻止生成正式施工资料，并提示“当前结果仅供预览，不建议作为正式交付材料”。
- 内置演示数据补充赛题要求的英文标准字段，同时保留原中文字段，避免破坏既有 BOM 与报告逻辑。

## 第 10 版：审查明细分组、纤芯分配表、批量文件与最近缓存

- 审查明细支持按不同类别切换查看，避免正常项和问题项全部堆在同一张表中。
- 审查明细按风险等级增加颜色提示：错误、高风险、提醒、正常分别使用不同底色。
- 智能转化结果新增“纤芯分配表”页签。
- 纤芯分配表可基于 `fiber_start`、`fiber_end`、`fiber_core_count` 自动展开纤芯编号。
- 若上传数据提供 `fiber_core_id` 或 `fiber_core_no`，优先使用现有纤芯编号。
- Excel 导出新增“纤芯分配表”Sheet。
- 上传控件支持一次选择多份 Excel/CSV 文件。
- 新增批量文件处理概览，显示每个文件的记录数、总体结论、错误/高风险/提醒/正常数量和处理状态。
- 新增最近 10 份文件缓存，缓存目录为 `.design_file_cache/`，并加入 `.gitignore`。
- 修复缓存功能首次接入时 `CACHE_INDEX_FILE` 未定义导致的运行时报错。

## 第 11 版：交接文档与规则文档固化

- 新增 `AGENTS.md`，固化项目目标、技术栈、运行命令、测试命令、禁止随意修改的方向和后续 Codex 工作规则。
- 新增 `docs/handoff.md`，记录当前工程目标、已完成功能、已知错误、未解决事项、下一轮最小任务和优先读取文件。
- 新增 `docs/task-log.md`，记录本轮关键操作、读取文件、修改文件、执行命令、成功结果、失败结果和待处理事项。
- 生成规则说明文档：
  - `docs/程序检查规则.docx`
  - `docs/数据校验规则.docx`
- 新增 `docs/build_rule_docs.py` 用于生成规则说明文档。
- 已知限制：本机未检测到 LibreOffice/soffice，因此 DOCX 渲染 QA 未完成。
- GitHub 已同步提交 `0364f96 Add design review and handoff docs`，提交内容包括核心代码、README、`.gitignore`、`AGENTS.md` 和交接文档。

## 第 12 版：批量审查溯源、规则文档输出与仓库同步规范

- 批量上传多份图纸/设计表时，轻量智能审查明细新增 `source_file` 字段，页面展示为“文件来源”列。
- 每条错误、高风险、提醒或正常记录均可追溯到对应上传文件，解决批量导入后无法判断是哪一张图纸报错的问题。
- 新增批量审查明细汇总逻辑，汇总所有上传文件的审查结果，而不仅展示当前选中文件。
- 审查明细筛选改为五类页签：
  - `全部明细`
  - `错误项`
  - `高风险项`
  - `提醒项`
  - `正常项`
- 生成并保留两份 Word 规则说明文档：
  - `docs/程序检查规则.docx`
  - `docs/数据校验规则.docx`
- 新增 `docs/build_rule_docs.py`，用于可复现生成上述规则说明文档。
- README 增补文件来源追踪、五类筛选、规则文档和版本范围说明。
- `AGENTS.md` 增补长期维护规则：后续每次项目更新都应同步更新 `CHANGELOG.md`，并在验证通过后提交、推送 GitHub 仓库。
- 本轮验证包括 `py_compile app.py` 和规则文档生成脚本语法检查；页面启动受 `.venv` Python 路径失效影响，备用启动可到达 Streamlit 启动日志但未完成独立 `200` 健康检查。
- 已知限制：DOCX 渲染 QA 仍受本机缺少 LibreOffice/soffice 影响，无法输出 PNG 视觉检查图。
